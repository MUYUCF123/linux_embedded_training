# Day1-Day2 学习总结

## 1. 我做了什么
### 重温指针

在开始该项目之前，我对指针十分陌生，停留在“地址”的模糊印象，于是进行了简单的入门回忆：

1. 可以让函数返回多个值，直接在指向的地址写值，就很灵活
2. 指针的运算，简单来说就是 p++，而加了多少就要看你的 p 指向了什么类型的变量，比如 char 是 1 字节，int 是 4 字节，long 是 4 字节，long long 则是 8 字节
3. 野指针和悬空指针，就是你指向的空间必定要有值，未分配和被释放都是不允许的。例如：在某个函数里定义一个指针，使其指向一个函数中的局部变量，但由于局部变量生命周期结束，指针便会变为悬空指针
4. 数组指针，先看问题：*void func(int \*p)* 以及*void func(int p[10])* 的区别，这里学到说叫数组退化，就是你传一个 p[10] 进函数，会丢失大小信息，相当于传入一个 p。

### 学习了 `sizeof` 和 `strlen`

手写了*strlen*函数，逻辑很简单，主要还是区别二者，*sizeof* 作为一个**运算符**，职责就是计算传入数据的大小，传入 int 变量你会得到 4，传入"hello"这样的字符串，你会得到 6（hello 以及结尾的 '\0'）；而 *strlen* 则是统计到 '\0' 之前

### 实现了 `my_strcpy`

依旧是手写简单函数练习，但学到了：

1. assert 断言，在该函数中，可以利用 *assert* 进行判断传入指针是否为空，例：**assert(dest != NULL)**
2. 函数返回值，之前一直觉得此类函数不需要返回值，但其实可以返回目标指针，方便链式调用，例如：*my_strcpy(dest, src)* 可以直接用在其他函数中，比如：*printf("%s", my_strcpy(dest, src))*
3. 更简洁的写法，之前的写法是：**while(*src != '\0') { *dest = *src; dest++; src++; } *dest = '\0';** 现在可以简化为：**while((\*dest++ = \*src++) != '\0');**

### 实现了 `my_strncpy`

简单练习，没什么好说的

### 实现了 `my_memcpy`

1. 学到了 void\* 适合通用接口，可以指向任何类型的内存，适合实现通用的内存操作函数，**但是**，void\*** 不能直接进行算术运算，需要先转换为 *char\***，因为 *char* 是字节级的指针，可以逐字节访问内存。

### 实现了 `reverse`

1. 反转字符串的思路是：交换首尾元素，逐渐向中间靠拢，直到指针相遇或交错。
